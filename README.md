# Онлайн платформа торговой сети электроники

Этот проект является онлайн платформой для управления торговой сетью электроники. Он включает в себя API интерфейс и административную панель.

## Технические требования

- Python 3.8+
- Django 3+
- Django REST Framework (DRF) 3.10+
- PostgreSQL 10+

## Установка и запуск

1. Клонируйте репозиторий:

   ```bash
   git clone ссылка_на_репозиторий
   cd название_репозитория

2. Установите зависимости:

   - pip install -r requirements.txt
   
   - poetry install (если используете Poetry)
3. Создайте базу данных и примените миграции:

   python manage.py migrate

4. Запустите сервер разработки:

   python manage.py runserver

   Приложение будет доступно по адресу http://localhost:8000/.

## Административная панель

Административная панель Django предоставляет удобный интерфейс для управления объектами сети. Для доступа к админ-панели необходимо создать суперпользователя:

python manage.py createsuperuser

После этого вы сможете получить доступ к административной панели по адресу http://localhost:8000/admin/.

## API

API интерфейс предоставляет возможность взаимодействия с данными с помощью HTTP запросов.

## Эндпоинты Network

### GET /networks/

- Описание: Получение списка всех звеньев сети.
- Endpoint: `/networks/`
- Ответ: Массив объектов, представляющих звенья сети.

### GET /networks/?country={country}

- Описание: Получение списка звеньев сети, фильтрованных по стране.
- Endpoint: /networks/
- Параметры запроса: country - название страны для фильтрации.
- Ответ: Массив объектов, представляющих звенья сети, отфильтрованных по указанной стране.

Пример использования:

      
      GET /networks/?country=USA

### POST /networks/

- Описание: Создание нового звена сети.
- Endpoint: `/networks/`
- Параметры запроса: Данные нового звена сети в формате JSON.
- Ответ: Созданный объект звена сети.

### GET /networks/{id}/

- Описание: Получение информации о конкретном звене сети.
- Endpoint: `/networks/{id}/`
- Параметры запроса: `id` - идентификатор звена сети.
- Ответ: Объект, представляющий звено сети с указанным `id`.

### PUT /networks/{id}/

- Описание: Обновление информации о звене сети с указанным `id`.
- Endpoint: `/networks/{id}/`
- Параметры запроса: `id` - идентификатор звена сети, данные обновления в формате JSON.
- Ответ: Обновленный объект звена сети.

### DELETE /networks/{id}/

- Описание: Удаление звена сети с указанным `id`.
- Endpoint: `/networks/{id}/`
- Параметры запроса: `id` - идентификатор звена сети.
- Ответ: Отсутствует (204 No Content).

## Эндпоинты аутентификации (Token)

### POST /token/

- Описание: Получение пары токенов доступа и обновления.
- Endpoint: `/token/`
- Параметры запроса: Данные пользователя для аутентификации (username, password) в формате JSON.
- Ответ: Объект с токенами доступа и обновления.

### POST /token/refresh/

- Описание: Обновление токена доступа.
- Endpoint: `/token/refresh/`
- Параметры запроса: Данные для обновления токена доступа (refresh) в формате JSON.
- Ответ: Обновленный токен доступа.
